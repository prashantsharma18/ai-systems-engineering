# Books

Annotated list of books worth reading. Notes reflect personal reading — updated as I work through each.

---

## Python

- **Fluent Python** — Luciano Ramalho. The definitive reference for Python internals. Chapters on data model, iterators, and concurrency are essential.
- **Architecture Patterns with Python** — Harry Percival & Bob Gregory. DDD and clean architecture applied to Python. Excellent for thinking about ML system design.

---

## Math for ML

- **Mathematics for Machine Learning** — Deisenroth, Faisal, Ong. Free online. Covers linear algebra, calculus, and probability with ML context. Good starting point.
- **The Elements of Statistical Learning** — Hastie, Tibshirani, Friedman. Free PDF. Dense but authoritative reference for classical ML theory.

---

## Machine Learning

- **Hands-On Machine Learning with Scikit-Learn, Keras and TensorFlow** — Aurélien Géron. Best practical ML book. Code-heavy, covers end-to-end workflows.
- **Designing Machine Learning Systems** — Chip Huyen. Essential for understanding how ML works as an engineering discipline, not just an algorithm discipline.

---

## Deep Learning

- **Deep Learning** — Goodfellow, Bengio, Courville. Free online. The theoretical reference. Read alongside implementation.
- **The Little Book of Deep Learning** — François Fleuret. Free PDF. Concise and surprisingly deep. Good for solidifying intuitions.

---

## LLMs and Generative AI

- **Building LLMs for Production** — Towards AI. Practical guide to LLM application architecture.

---

## MLOps and Systems

- **Reliable Machine Learning** — Orenstein et al. O'Reilly. Covers production ML reliability, monitoring, and incident response.
